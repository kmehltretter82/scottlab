Finite Scott information systems: a concise reference

This page states the definitions ScottLab teaches interactively. It follows Dana S. Scott, “Domains for Denotational Semantics” (1982), with the explicit distinguished token Δ.

Information systems

An information system is a tuple A = (T, Δ, Con, ⊢):

  • T — a finite set of tokens (observations);
  • Δ ∈ T — the distinguished, always-present token;
  • Con — the finite consistent subsets of T;
  • ⊢ — entailment, relating a consistent finite set to a token.

The laws (whenever the displayed entailments are defined):

1. X ⊆ Y ∈ Con            ⇒ X ∈ Con        (downward consistency)
2. {a} ∈ Con                                (singleton consistency)
3. X ∈ Con and X ⊢ a       ⇒ X ∪ {a} ∈ Con  (entailment preserves Con)
4. X ∈ Con                 ⇒ X ⊢ Δ          (distinguished token)
5. a ∈ X ∈ Con             ⇒ X ⊢ a          (reflexivity)
6. Y ⊢ X and X ⊢ a         ⇒ Y ⊢ a          (cut)

Most later literature (Larsen–Winskel 1984; Winskel 1993, ch. 12) drops Δ and uses the triple (T, Con, ⊢). The two presentations generate isomorphic domains; in the Δ-free form the least state is {a | ∅ ⊢ a}.

Closure, states, bottom

For consistent S: closure(S) = { a ∈ T | S ⊢ a }. A state is a consistent, entailment-closed token set. States ordered by inclusion form the information order; its least element is ⊥ = closure(∅) = closure({Δ}). Δ is a token inside ⊥; it is not another name for ⊥.

Approximable mappings

A finite-generator mapping from A to B is a set of rules P ⇒ b (finite consistent P ⊆ T_A, token b ∈ T_B). Applied to a state x: 

F(x) = closure_B({ b | some P ⇒ b has P ⊆ x })

Such maps are monotone, and — because every conclusion needs only a finite premise — Scott continuous. The states of the function-space system A → B are exactly these mappings (Larsen–Winskel, Prop. 4.20); for the flat Booleans, [Bool → Bool] has eleven states whose four maximal points are the four total Boolean functions.

Least fixed points

For an endomapping F on a finite system, iterate from bottom: ⊥ ⊑ F(⊥) ⊑ F²(⊥) ⊑ … The chain ascends by monotonicity and must repeat; the first repeated state is the least fixed point (token-level version: Scott, PRG-19, Theorem 4.1). On a finite domain, iteration stabilizes within the domain's height. Bounded systems truncate honestly: every displayed iterate is a genuine finite iterate of the unbounded object.

Computational limits

ScottLab's validator checks the laws exhaustively over all consistent subsets — Θ(2ⁿ) in the token count n, run once per system and cached. Mapping validation carries an explicit subset-check budget (default 262,144), and the gallery asks before enumerating states of systems with more than 12 tokens. Teaching-scale systems stay well inside these limits.
